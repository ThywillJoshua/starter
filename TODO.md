 Component palette (from `ui/src/app/features/docs/widgets` and renderer tokens)
- Callout (variants: info, success, warning, danger, note; supports custom icon/color).
- Steps (ordered walkthrough with titles, optional icons/numbers).
- Tabs (horizontal/vertical content toggles).
- Card, CardGroup, CardGrid (link hubs or option choices; support icon/img/cta).
- Accordion (folded FAQs/workflows), Tooltip (inline hints).
- Frame/Embed (screenshots, GIF-like videos, external embeds).
- Files (links to attachments), Fields/Responses (for API-ish specs if needed).

 Enrichment patterns
- Use a consistent flow per article: quick intro → prerequisites (Callout) → numbered Steps → troubleshooting/FAQs (Accordion or Callouts) → links to related pages (CardGroup/CardGrid).
- GIF/video capture: prefer `<Frame>` embedding MP4/WebM or GIF; keep clips under 20s, file names kebab-cased by slug (e.g., `images/team-feedback/skip-a-question.mp4`); add a note callout when a recording is still needed.
- Screenshots: store alongside clips in `pulse-docs/images/<area>/` with descriptive names; reference via `<Frame src="/images/<area>/<file>.png" />`.
- Accessibility: always add alt text/captions on Frame, keep headings under 60 chars, and use actionable link text.
- Tone/voice: concise, practical, and direct; prefer action verbs; keep jargon minimal; always include who/when for steps; CTAs use verbs (“Open survey settings”, “Preview email”) not generic “Click here”.
